# Translation System Rework Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use compose:subagent (recommended) or compose:execute to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the CSS show/hide i18n system with a key-based translation system where English is pre-rendered and other languages are delivered via a translations JSON fetched at runtime.

**Architecture:** English HTML is pre-rendered at build time using a custom `{{t 'key'}}` Handlebars helper that reads from `translations.json`. Each translatable element gets a `data-i18n="key"` attribute. At runtime, a language switcher JS fetches `translations.json` and swaps text content using those keys. Component data.js files retain their `{en, ru, uk}` objects with translation keys. Only one language exists in the DOM at any time.

**Tech Stack:** Handlebars (custom helper), SCSS, vanilla JavaScript (fetch), TARS user-task (gulp), JSON

## Translation Key Convention

Hierarchical keys using dot notation: `section.element.variant`

| Component | Key Pattern | Example |
|-----------|-------------|---------|
| Header | `header.name`, `header.title1`, `header.githubLabel` | `header.name` → "Denis" |
| Section titles | `section.skills.title`, `section.experience.title` | `section.skills.title` → "Skills" |
| Data items | `skills.{area}.{index}.{field}` | `skills.magento2.0.title` → "XML Layouts" |
| Footer | `footer.name`, `footer.offer` | `footer.offer` → "Open to suggestions" |

### Task 2: Register Handlebars `{{t}}` Helper

**Files:**
- Modify: `tars/user-tasks/html/helpers/handlebars-helpers.js`

**Interfaces:**
- Produces: `{{t 'header.name'}}` helper — looks up key in translations.json for current language
- Produces: `{{langCode}}` helper — returns current language code
- Consumes: `translations` and `lang` from template context (set by page data)

- [ ] **Step 1: Replace handlebars-helpers.js**

Replace `tars/user-tasks/html/helpers/handlebars-helpers.js` with:

```js
'use strict';

const Handlebars = tars.packages.handlebars;
const fs = require('fs');
const path = require('path');

const translationsPath = path.resolve(process.cwd(), 'markup/translations.json');
let translations = {};

try {
    translations = JSON.parse(fs.readFileSync(translationsPath, 'utf8'));
} catch (e) {
    console.warn('Could not load translations.json:', e.message);
}

let currentLang = 'en';

function getNestedValue(obj, keyPath) {
    return keyPath.split('.').reduce((acc, part) => {
        if (acc && typeof acc === 'object') {
            return acc[part];
        }
        return undefined;
    }, obj);
}

const handlebarsHelpers = {
    t(key) {
        if (!key || typeof key !== 'string') {
            return '';
        }

        const lang = currentLang;
        const langData = translations[lang];

        if (!langData) {
            return new Handlebars.SafeString(`[missing lang: ${lang}]`);
        }

        const value = getNestedValue(langData, key);

        if (value === undefined) {
            return new Handlebars.SafeString(`[missing: ${key}]`);
        }

        return new Handlebars.SafeString(value);
    },

    langCode() {
        return currentLang;
    },

    setLang(lang) {
        if (typeof lang === 'string') {
            currentLang = lang;
        }
        return '';
    },

    langSelect(options) {
        const hash = options.hash;
        if (hash[currentLang] !== undefined) {
            return new Handlebars.SafeString(hash[currentLang]);
        }
        if (hash.default !== undefined) {
            return new Handlebars.SafeString(hash.default);
        }
        return '';
    }
};

module.exports = handlebarsHelpers;
```

### Task 6: Create Language Switcher JS

**Files:**
- Create: `markup/components/header/language-switcher.js`

**Interfaces:**
- Fetches `translations.json` from the same directory as the page
- Reads `data-i18n` attributes to find translatable elements
- Swaps text content with translated values
- Persists language to localStorage
- Updates `<html lang>` attribute
- No page reload

- [ ] **Step 1: Create language-switcher.js**

```js
(() => {
    const STORAGE_KEY = 'preferred-lang';
    const LANGS = ['en', 'ru', 'uk'];
    const DEFAULT_LANG = 'en';

    let translations = null;
    let currentLang = DEFAULT_LANG;

    const toggle = document.querySelector('.lang-toggle');

    if (!toggle) {
        return;
    }

    function getNestedValue(obj, keyPath) {
        return keyPath.split('.').reduce((acc, part) => {
            if (acc && typeof acc === 'object') {
                return acc[part];
            }
            return undefined;
        }, obj);
    }

    async function loadTranslations() {
        if (translations) {
            return translations;
        }

        try {
            const response = await fetch('translations.json');
            translations = await response.json();
            return translations;
        } catch (error) {
            console.error('Failed to load translations:', error);
            return null;
        }
    }

    function applyTranslations(lang) {
        if (!translations || !translations[lang]) {
            return;
        }

        const langData = translations[lang];

        document.querySelectorAll('[data-i18n]').forEach((element) => {
            const key = element.getAttribute('data-i18n');
            const value = getNestedValue(langData, key);

            if (value !== undefined) {
                element.textContent = value;
            }
        });

        document.querySelectorAll('[data-i18n-attr]').forEach((element) => {
            const pairs = element.getAttribute('data-i18n-attr').split(';');

            pairs.forEach((pair) => {
                const [attr, key] = pair.split(':');

                if (attr && key) {
                    const value = getNestedValue(langData, key.trim());

                    if (value !== undefined) {
                        element.setAttribute(attr.trim(), value);
                    }
                }
            });
        });

        document.documentElement.lang = lang;
        currentLang = lang;
        localStorage.setItem(STORAGE_KEY, lang);

        const langText = toggle.querySelector('.lang-toggle-text');

        if (langText) {
            langText.textContent = lang;
        }

        const pdfLink = document.querySelector('a[href*="Resume-"]');

        if (pdfLink) {
            pdfLink.href = pdfLink.href.replace(/Resume-\w+\.pdf/, `Resume-${lang}.pdf`);
        }
    }

    async function switchLanguage(lang) {
        if (lang === currentLang) {
            return;
        }

        if (!translations) {
            await loadTranslations();
        }

        if (translations && translations[lang]) {
            applyTranslations(lang);
        }
    }

    function getNextLang() {
        const index = LANGS.indexOf(currentLang);
        return LANGS[(index + 1) % LANGS.length];
    }

    toggle.addEventListener('click', () => {
        const next = getNextLang();
        switchLanguage(next);
    });

    const savedLang = localStorage.getItem(STORAGE_KEY);

    if (savedLang && LANGS.includes(savedLang) && savedLang !== DEFAULT_LANG) {
        loadTranslations().then(() => {
            if (translations && translations[savedLang]) {
                applyTranslations(savedLang);
            }
        });
    }
})();
```

## How It Works

### Build Time
1. TARS compiles `markup/pages/index.html` with all component data
2. `{{t 'header.name'}}` helper reads from `translations.json` → outputs English text
3. `data-i18n="header.name"` attribute is added to each translatable element
4. `{{langCode}}` outputs `'en'` for PDF links and toggle text
5. User-task `build-translations` copies `translations.json` to `dev/`

### Runtime (English — default, no JS needed)
1. Page loads with English text pre-rendered in HTML
2. No JS required for initial display
3. `data-i18n` attributes are inert (just metadata)

### Runtime (Language switch)
1. User clicks language toggle → `language-switcher.js` fires
2. JS checks localStorage for saved preference on page load
3. If non-English preferred: fetch `translations.json`, apply translations
4. On toggle click: load translations (if not cached), swap text content
5. Updates `<html lang>`, PDF link href, localStorage
6. Only English text is in DOM initially; after switch, only target language

### data-i18n Attribute Protocol
- `data-i18n="key"` — replaces element's `textContent` with `translations[lang].key`
- `data-i18n-attr="aria-label:key;title:key2"` — replaces attribute values (semicolon-separated pairs)
