# Fix CLS on Non-English Page Load Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use compose:subagent (recommended) or compose:execute to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate CLS when loading with a saved non-English language preference by hiding the page until translations are applied.

**Architecture:** CSS hides `<body>` until `[data-lang-ready]` attribute is set on `<html>`. An inline `<head>` script checks localStorage — for English, it reveals immediately (no fetch needed). For non-English, it sets a flag that the language-switcher reads. The switcher fetches translations, applies them, then reveals the page. English users see zero delay; non-English users see a brief blank instead of a layout shift.

**Tech Stack:** CSS, vanilla JavaScript (inline script + existing language-switcher.js)

- [ ] **Step 1: Add inline script and CSS to index.html**

Edit `markup/pages/index.html`. Replace the `<html>` tag and add after the theme-switcher include:

```handlebars
<!doctype html>
<html class="no-js" lang="en">

<head>
    {{> head/head head.defaults}}
    {{> before-page-content/theme-switcher}}
    <style>body:not([data-lang-ready]){visibility:hidden}</style>
    <script>
        (function() {
            var KEY = 'preferred-lang';
            var saved = localStorage.getItem(KEY);
            if (!saved || saved === 'en') {
                document.documentElement.setAttribute('data-lang-ready', '');
            } else {
                window.__pendingLang = saved;
            }
        })();
    </script>
</head>
```

- [ ] **Step 2: Update language-switcher.js to reveal page**

Edit `markup/components/header/language-switcher.js`. Add at the top of the IIFE (after the constants):

```js
    function revealPage() {
        document.documentElement.setAttribute('data-lang-ready', '');
    }
```

Replace the saved language block at the bottom (lines 112-120) with:

```js
    const savedLang = window.__pendingLang || localStorage.getItem(STORAGE_KEY);

    if (savedLang && LANGS.includes(savedLang) && savedLang !== DEFAULT_LANG) {
        loadTranslations().then(() => {
            if (translations && translations[savedLang]) {
                applyTranslations(savedLang);
            }
            revealPage();
        });
    } else {
        revealPage();
    }
```

## How It Works

| Scenario | Flow | Visual |
|---|---|---|
| English (default) | Inline script sets `data-lang-ready` immediately → body visible | No delay |
| English (saved) | Inline script sets `data-lang-ready` immediately → body visible | No delay |
| Non-English (saved) | Inline script sets `window.__pendingLang` → body hidden → switcher fetches translations → applies → reveals | Brief blank, no CLS |
| Non-English (first visit) | No saved preference → English shown → user clicks toggle → switcher fetches → swaps | Normal switch |

**Why no CLS:** The user never sees English text when they have a non-English preference. The body is hidden until the correct language is applied. Text length differences (Russian ~30% longer than English) don't cause shifts because the layout is computed with the final text.
